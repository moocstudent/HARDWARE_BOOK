## Core Notes

### The oldest, most common serial link

**UART (Universal Asynchronous Receiver/Transmitter)** is the most basic way chips talk, and it's what runs behind uploading an Arduino sketch or printing debug output with `Serial.println`. It's **point-to-point**: two devices, two data wires, back and forth.

- **TX (Transmit):** send
- **RX (Receive):** receive

The wiring rule is **cross-over**: one side's TX goes to the other's RX and vice versa, plus a **common ground (GND)** as the shared reference.

```text
  A.TX ──────► B.RX
  A.RX ◄────── B.TX
  A.GND ────── B.GND
```

### Asynchronous: no clock line

The **A (Asynchronous)** in UART is the crux: the two sides **share no clock line**. So how does the receiver know how long each bit lasts? By **agreeing on a rate in advance** — the **baud rate**.

Baud rate is bits per second. Common values: 9600, 115200… As long as both sides are set to the **same** baud rate, the receiver can sample each bit at the right instant to that beat.

> bit time = 1 / baud. At 9600 baud, each bit is ~104 µs.

**Mismatched baud is UART's most common failure**: garbage in the serial monitor is nearly always a wrong baud setting.

### The structure of a frame

At rest the line stays HIGH. To send a byte, the data is packed into a **frame**:

1. **Start bit:** one LOW bit that tells the receiver "here it comes" and lets it align its timing.
2. **Data bits:** usually 8, **LSB first**.
3. **Parity bit:** an optional single bit for simple error checking.
4. **Stop bit:** one (or two) HIGH bits marking the end of the frame.

In the interactive figure, drag the byte to send and watch the start bit, 8 data bits and stop bit lay out on the line. The familiar "8N1" means 8 data bits, No parity, 1 stop bit.

### Talking with Serial

```cpp
void setup() {
  Serial.begin(9600);        // set the baud rate
}
void loop() {
  Serial.println("Hello");   // send a line
  if (Serial.available()) {  // anything received?
    char c = Serial.read();  // read one byte
    Serial.print("got: ");
    Serial.println(c);
  }
  delay(500);
}
```

> ⚠ **Key point:** UART is an asynchronous point-to-point serial link — no clock line, synced by an agreed **baud rate**. Remember three things: **cross TX/RX**, **share ground**, **match the baud**. Garbage almost always means one of these three is wrong. When bridging 5V and 3.3V parts, don't forget level matching.
