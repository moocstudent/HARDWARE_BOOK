## 核心讲义

### 两类显示,一个共同问题

给项目加一块屏,大致有两类选择:

- **字符屏(LCD1602):** 只能显示预设的字符,分 16 列 × 2 行。简单、便宜、字大。
- **像素屏(SSD1306 OLED):** 由一个个像素点组成(常见 128×64),能画文字、图形、进度条,自发光、对比度高。

无论哪种,核心问题都一样:**数据怎么送进屏?** 答案就是前面学过的总线。

### LCD1602:字符与并口

LCD1602 内部有一张字模表,你送进去一个字符编码,它就在对应格子显示那个字。传统接法用**并口**——4 或 8 根数据线加几根控制线,接线很多。

好在有 **I²C 背包(backpack)**:一块小转接板焊在屏后,把十几根线**压缩成 4 根**(VCC、GND、SDA、SCL),用 I²C(见 BUS2)通信:

```cpp
#include <LiquidCrystal_I2C.h>
LiquidCrystal_I2C lcd(0x27, 16, 2);   // 地址常为 0x27 或 0x3F
void setup() {
  lcd.init(); lcd.backlight();
  lcd.setCursor(0, 0); lcd.print("Hello, Hardware");
  lcd.setCursor(0, 1); lcd.print("Line 2");
}
void loop() {}
```

### SSD1306 OLED:像素与显存

SSD1306 是像素屏,内部有一块**显存(frame buffer)**——屏上每个像素对应显存里的一个比特。你在库里「画」的一切(文字、线、图)先写进这块缓冲,再一次性 **刷新(display)** 到屏上。它通常也走 I²C(4 根线),地址常为 0x3C。

```cpp
#include <Adafruit_SSD1306.h>
Adafruit_SSD1306 oled(128, 64, &Wire);
void setup() {
  oled.begin(SSD1306_SWITCHCAPVCC, 0x3C);
  oled.clearDisplay();
  oled.setTextSize(1); oled.setTextColor(SSD1306_WHITE);
  oled.setCursor(0, 0); oled.println("Hello OLED");
  oled.display();               // 把缓冲刷到屏上
}
void loop() {}
```

### 显示库的通用套路

几乎所有显示库都遵循同一节奏:**初始化 → 清屏 → 设定位置/大小/颜色 → 写内容 → (像素屏)刷新**。理解了「先写显存、再刷新」这一步,你就能上手绝大多数屏。

> ✦ **要点:** 字符屏送字符编码、像素屏写显存;两者常用 **I²C 只占 4 根线**。像素屏记得画完要 `display()` 刷新。屏不亮先查:I²C 地址对不对、上拉电阻有没有、初始化调了没。
