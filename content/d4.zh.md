## 核心讲义

### 现实是连续的,芯片是离散的

温度、光照、旋钮位置……现实世界的量是**连续**的。但单片机内部只有 0 和 1。要让芯片「读懂」一个模拟电压,需要一个**模数转换器(ADC,Analog-to-Digital Converter)**,把连续的电压**量化**成一个整数。

**量化**就是把一段电压范围切成许多等高的台阶,然后报告输入落在第几级。台阶越多(位数越高),还原得越精细。

### 分辨率:10 位 = 0–1023

ADC 的**位数**决定台阶数量。Arduino Uno 的 ADC 是 **10 位**,即 2¹⁰ = **1024** 个等级,读数范围 **0–1023**:

- 读数 0 ↔ 0 V
- 读数 1023 ↔ 参考电压(Uno 默认 5 V)
- 每一级 ≈ 5V / 1024 ≈ 4.9 mV

### analogRead 与电压换算

`analogRead(pin)` 一步读出量化结果:

```cpp
int raw = analogRead(A0);          // 0..1023
float volts = raw * 5.0 / 1023.0;  // 换算回电压
```

上方交互图拖动输入电压,可看到阶梯状的转换曲线,以及读数如何随电压跳级上升。

配合 CF3 的分压器,读一个电位器:

```cpp
void setup() { Serial.begin(9600); }
void loop() {
  int raw = analogRead(A0);
  float v = raw * 5.0 / 1023.0;
  Serial.print(raw); Serial.print("  "); Serial.println(v);
  delay(200);
}
```

### 参考电压与采样率

- **参考电压 Vref:** 读数 1023 对应的电压。默认是供电 5V,但也能用更稳定的外部/内部基准提高精度。**换算时必须用实际 Vref**,否则电压算错。
- **采样率:** ADC 转换一次需要时间(Uno 上约 100 µs)。读得太频繁、或信号变化太快,都会失真。读慢变量(温度、旋钮)绰绰有余;读音频等高频信号则要看芯片的采样能力。

不同板子位数不同:

| 板子 | ADC 位数 | 满量程读数 | 逻辑电压 |
|---|---|---|---|
| Arduino Uno | 10 位 | 0–1023 | 5 V |
| ESP32 | 12 位 | 0–4095 | 3.3 V |
| Pi Pico | 12 位 | 0–4095 | 3.3 V |

> ✦ **要点:** ADC = 量化。读数 = round(Vin/Vref × (级数−1))。要还原成电压,乘回 Vref 再除以满量程。位数越高越精细,但换算逻辑不变。
