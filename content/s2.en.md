## Core Notes

### One wire can carry data too

The DHT22 (a.k.a. AM2302) is a temperature/humidity sensor that sends both readings over a **single data wire**. It already converts the analog quantities to digital internally, so you don't wire an ADC — but the price is a **custom one-wire timing protocol**: sender and receiver "tap Morse code" on that wire using agreed HIGH/LOW durations.

### The 40-bit frame and its checksum

The MCU first pulls the data line LOW for a while to "request"; after the DHT22 responds it streams out **40 bits** at once:

| Field | Bits | Meaning |
|---|---|---|
| Humidity high byte | 8 | integer part of humidity |
| Humidity low byte | 8 | fractional part |
| Temperature high byte | 8 | integer part of temperature |
| Temperature low byte | 8 | fractional part |
| Checksum | 8 | low 8 bits of the sum of the four bytes |

Whether a bit is 0 or 1 depends on **how long it stays HIGH** (short = 0, long = 1). The final byte is a **checksum**: add the first four bytes; the low 8 bits should equal it. If they don't, the read was disturbed and should be discarded.

### Reading with a library

Bit-banging this timing by hand is fiddly; in practice everyone uses a ready-made library (e.g. the DHT sensor library):

```cpp
#include <DHT.h>
#define DHTPIN 2
#define DHTTYPE DHT22
DHT dht(DHTPIN, DHTTYPE);

void setup() { Serial.begin(9600); dht.begin(); }
void loop() {
  float h = dht.readHumidity();
  float t = dht.readTemperature();
  if (isnan(h) || isnan(t)) { Serial.println("read failed"); }
  else {
    Serial.print(t); Serial.print("C  ");
    Serial.print(h); Serial.println("%");
  }
  delay(2000);   // DHT22 needs at least 2 s between reads
}
```

### Two common pitfalls

- **Don't sample too fast:** the DHT22 needs at least **2 seconds** between reads; reading more often returns stale data or fails.
- **Pull-up resistor:** the data line usually needs a ~**10 kΩ pull-up** to Vcc (many breakout boards include it).

> ✦ **Key point:** the DHT22 streams 40 bits (humidity + temperature + checksum) over one wire using **one-wire timing**. Understand the principle, but use a library in practice. Remember: **2-second spacing, pull up the data line, validate with the checksum**. These "digital sensors with their own protocol" are everywhere in embedded work.
