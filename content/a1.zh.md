## 核心讲义

### 舵机:你只管发角度

普通电机通电就一直转,你很难让它精确停在某个角度。**舵机(servo)**不一样:它内部集成了一个直流电机、一组减速齿轮、一个测角度的电位器和一块控制电路。你只要告诉它「我要 90°」,它就会自己转过去并**保持**在那里。这种「发角度即到位」的能力,来自内部的**闭环反馈**。

常见的爱好级舵机(如 SG90)能转约 0°–180°。

### 脉宽即角度

舵机听的不是电压,而是一串**脉冲的宽度**。它期待每 **20 ms(50 Hz)**收到一个脉冲,脉冲高电平持续的时间决定目标角度:

| 脉宽 | 角度 |
|---|---|
| 1.0 ms | 0° |
| 1.5 ms | 90°(中位) |
| 2.0 ms | 180° |

> 脉宽在 1–2 ms 之间线性对应 0°–180°。

上方交互图拖动角度,左边的脉冲高电平会随之变宽,右边的舵臂随之转动。

### 别把它和调光 PWM 弄混

这里有个容易混淆的点:舵机信号**也是**一种 PWM,但和 IO3 里调 LED 亮度的 PWM 关注点不同——

- **调光 PWM:** 看重**占空比**(平均值),频率不那么关键。
- **舵机 PWM:** 频率固定 **50 Hz**,看重的是**高电平的绝对时间**(1–2 ms),而不是占空比。

正因如此,直接用 `analogWrite` 很难精确控制舵机,应该用专门的 `Servo` 库,它在后台生成精准的 1–2 ms 脉冲。

### 用 Servo 库

```cpp
#include <Servo.h>
Servo myservo;

void setup() {
  myservo.attach(9);      // 信号线接到脚 9
}
void loop() {
  myservo.write(0);    delay(1000);   // 转到 0°
  myservo.write(90);   delay(1000);   // 转到 90°
  myservo.write(180);  delay(1000);   // 转到 180°
}
```

`write(angle)` 传入 0–180 的角度,库自动换算成脉宽。

舵机(棕/红/橙三线:GND / +5V / 信号)转动时电流不小,起步可达几百毫安。**不要用单片机的 5V 脚直接供电**,否则电压瞬间被拉低,板子可能反复重启。正确做法:舵机电源用独立的 5V,但**一定要和单片机共地(GND 相连)**,否则信号没有共同参考点,舵机会乱抖。

> ⚠ **要点:** 舵机 = 电机 + 齿轮 + 反馈,你发的是「角度」而非「转速」。控制信号是固定 50 Hz、1–2 ms 的脉宽。记住两条铁律:用 `Servo` 库、给它独立供电并共地。
