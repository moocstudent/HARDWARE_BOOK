## 核心讲义

### 两条互补的线索:压力图 + 姿态

「病人翻身了吗?离床了吗?体位对不对?」——要可靠地回答,靠单一传感器都不够,得用两条**互补**的线索交叉印证:

- **压力分布**——身体压在床垫上的「热力图」。哪里受压重、哪里悬空,一张图就看得清:能判断人在不在床、躺成什么形状、是否长时间死压同一处(压疮风险),甚至能定位到肩、臀、脚跟这些高危受压点。
- **惯性姿态**——一个贴在身上或床板上的 IMU,直接给出「朝向」与「运动」。它不知道身体压在哪,但它知道身体是仰是侧、有没有在动。

为什么要两条线索?因为各有盲区:压力图能看出「形状」却看不出细微的「朝向」;IMU 能测朝向却不知道人是否已经离床。**融合**两者,判断才既灵敏又不误报——这正是本项目「多传感器交叉验证」哲学的又一次体现。

### 力敏电阻:压得越重,电阻越小

压力图的基础器件是**力敏电阻(FSR,Force-Sensing Resistor)**或 **Velostat 导电薄膜**。它们的原理是「压力→电阻」:没受压时电阻很大(兆欧级),受压越大,内部导电颗粒接触越紧密,电阻越小(可降到几百欧)。

怎么把电阻变化读成电压?正是 CF3 学过的**分压**:把 FSR 和一个固定电阻串联,从中间抽头,用 ADC(见 IO4)读电压。压力大→FSR 阻值小→抽头电压变化,ADC 就读出压力的大小。

器件选择上有两条路:

- **现成方案:Interlink FSR 402 / 406** 圆形力敏电阻,一致性好、寿命长,适合定点测量(如床脚、报警按钮)。
- **低成本 DIY:Velostat 薄膜 + 铜箔胶带**做成行列交叉的网格。上下两层铜箔条正交排列,中间夹一层 Velostat,每个交叉点就是一个压力测点。用这个方法可以自制一整张覆盖床面的**压力垫**,成本极低,非常适合原型验证。

上方交互图就是一张这样的压力图:拖动「体位」滑块,压力最高的**肩部与臀部**两团色块会随身体左右偏移(侧卧时压力集中到一侧);点「模拟离床」则整张图归零——这正是离床报警的直接依据。注意臀部通常是压力最高、也是压疮最高发的部位。

### 用多路复用器扫描网格

一张压力垫有很多个测点。比如一个 12×6 的网格就有 72 个点,而 ESP32 或 Arduino 的 ADC 通道只有几路——不可能每个点接一根线。解决办法是**模拟多路复用器(analog multiplexer)**,它像一个「电子旋转开关」:

- **CD74HC4067** 是常见的 16 通道模拟开关。用 **4 根地址线**(2⁴ = 16)选择「此刻把 16 路输入中的哪一路接到公共输出」,公共输出再接到 ADC。
- 扫描时,单片机快速地把地址从 0 变到 15,逐一把每一路接通、读一次 ADC,就「扫」完了 16 个点。测点更多就并联多片,或用地址线扩展。

对行列网格,常见做法是**逐行供电、逐列读取**:给第 1 行加电压,依次读各列的电压(即第 1 行各点的压力),再切到第 2 行……扫完所有行就得到整张压力图。这和 LED 点阵、键盘矩阵的扫描是同一套思路。

### IMU:加速度计 + 陀螺仪 + 姿态融合

第二条线索来自 **IMU(惯性测量单元)**。它把两三种传感器集成在一颗芯片里,通过 **I²C**(见 BUS2)输出:

- **加速度计(accelerometer)**:测三个轴向的加速度。静止时它感受到的是重力,因此能算出芯片相对水平面的**倾角**——这就是「朝上/朝左/朝右」的来源。
- **陀螺仪(gyroscope)**:测三个轴的**角速度**(转得多快),对时间积分能得到转过的角度。它擅长捕捉「翻身」这种转动过程。
- **磁力计(magnetometer)**(9 轴 IMU 才有):像指南针,提供绝对方位参考。

| 器件 | 特点 |
|---|---|
| **MPU-6050** | 6 轴(加速度 + 陀螺),经济型,需自己做姿态融合 |
| **ICM-20948 / BNO055** | 9 轴,**芯片内自带姿态融合**,直接输出稳定的朝向 |

加速度计单用会被体动干扰,陀螺仪单用会随时间**漂移**(积分误差累积)。把两者数据用算法(如互补滤波、卡尔曼滤波)**融合**,才能得到既稳定又灵敏的朝向估计。MPU-6050 便宜但要你自己在树莓派上做融合;BNO055 内部已经算好了「四元数 / 欧拉角」,直接读就是稳定朝向,省心但贵一些。

用 IMU 判断:**翻身** = 朝向发生了一次持续的、超过阈值的变化(而不是短暂抖动);**离床前兆** = 检测到坐起、大幅加速度;配合压力图突然清零,就能高置信度地判定离床。

### 融合成有用的护理判断

把压力图与 IMU 两路信息交给树莓派融合,软件就能给出真正有临床意义的结论:

- **翻身计时器**:记录「距上次体位改变过了多久」。若超过设定时限(常见 2 小时),提醒护理人员为病人翻身,预防**压疮**。这是本子系统最核心的价值。
- **离床/跌倒报警**:压力图突然整体清零,或 IMU 检测到剧烈运动后压力消失 → 病人可能起身或跌落 → 立即报警。对失能、术后病人尤其重要。
- **体位识别**:结合压力分布形状与 IMU 朝向,分类为仰卧 / 左侧卧 / 右侧卧 / 俯卧 / 坐起。
- **压疮风险图**:累计每个受压点的「受压时长 × 压强」,给出高风险部位提示。

### 现实中的坑

- **FSR 一致性差**:同一批 FSR 阻值可能差很多,DIY 的 Velostat 网格更是每个点特性不一。要么逐点标定,要么只用相对/阈值判断(判断「有没有压」比「精确多少帕」更可靠)。
- **蠕变与滞后**:FSR 长时间受压,读数会缓慢变化(蠕变);压力去除后也不立即回零(滞后)。对「翻身检测」这类看变化的应用影响不大,但做绝对压力测量要注意。
- **扫描串扰**:行列网格扫描时,未选中的通路可能漏电流形成「鬼影」。加二极管或用有源扫描可缓解。
- **走线与耐用**:床面反复受压、清洁,柔性网格的导线容易疲劳断裂,要选耐弯折的材料并做好固定。

### 压疮:这个子系统真正要防的病

要理解这个子系统的价值,得先理解它要防的那个病。**压疮(压力性损伤,俗称褥疮)**是长期卧床病人最常见、也最能反映护理质量的并发症。成因很直接:身体某处长时间受压 → 该处毛细血管被压闭 → 组织缺血缺氧 → 细胞坏死。它有两个关键变量:**压强的大小**和**受压的时长**。低压持续很久,和高压持续较短,都可能致伤。

高发部位正是骨头突出、软组织薄的地方:**骶尾部(臀部)、脚跟、髋部、肩胛、后脑**。这也是为什么 CB3 的交互图把肩、臀画成压力最高的两团——它们既是承重点,也是压疮高危点。临床上防压疮的黄金法则就是**定时翻身**(常见每 2 小时一次),而「距上次翻身多久了」这个看似简单的计时,正是本子系统对护理最直接的贡献:它把「凭记忆和经验」变成「有数据、会提醒」。

### 定点 FSR vs 全床压力垫:两种粒度

压力感知有两种做法,粒度和成本差别很大:

- **少量定点 FSR**:在几个关键部位(如骶尾、脚跟)各放一个 FSR。便宜、简单,能回答「这个高危点受压多久了」,但看不到整体分布。
- **全床压力垫(密集网格)**:整张床面铺满测点,能画出完整的压力热力图,精确定位、算压力中心、判断体位形状。信息丰富但成本高、走线复杂、需要多路复用扫描(见上文)。

原型阶段常从「定点 FSR + DIY 稀疏网格」起步验证,产品阶段再上商用校准压力垫(Tekscan / XSensor)。这又是一次「先便宜跑通、再升级」的体现。

### 压力中心(CoP):从一堆数字到一个判断

有了整张压力图,怎么从几十个数值得出「体位」这样的判断?一个常用的中间量是**压力中心(Center of Pressure, CoP)**——把所有测点的压力当作权重,算出加权平均位置,相当于「身体压力的重心在床面的投影」。

- CoP 居中、分布对称 → 仰卧;
- CoP 偏向一侧、压力集中 → 侧卧(偏哪侧就是哪侧卧);
- CoP 位置随时间发生一次明显、持续的迁移 → 发生了一次翻身;
- CoP 消失(总压力骤降到接近零)→ 离床。

把高维的压力图**降维**成 CoP 这样一两个好懂的量,再结合 IMU 的朝向,判断就既简单又稳健。这是传感器数据处理里非常通用的思路:先提取有物理意义的特征,再在特征上做判断。

### 加速度计怎么就测出了倾角

很多人好奇:一个测「加速度」的芯片,怎么知道床上的人是仰是侧?关键在于**重力**。加速度计静止时,它并不读到零,而是读到**重力加速度 g** 沿三个轴的分量。芯片平放时,g 全落在 Z 轴;芯片立起来,g 就转移到别的轴。通过三个轴上 g 的分配比例,用反三角函数就能算出芯片相对水平面的**倾角**。

所以贴在病人胸前或床板上的 IMU,靠的就是「重力往哪个轴偏」来判断朝向。注意这招只在**相对静止**时准——一旦有运动加速度混进来,就分不清哪部分是重力、哪部分是运动了,这正是需要陀螺仪来补的地方(运动时信陀螺,静止时信加速度计,融合起来)。

### 现实中的坑(补充)

- **阈值不是常数**:不同病人体重、床垫软硬不同,同样的「翻身」在压力图上的变化幅度差别很大。好的系统要能自适应校准,或用相对变化而非固定阈值。
- **被子与床品**:厚被子会分散压力、模糊压力图;IMU 若贴在被子外则测的是被子不是人。安装位置要想清楚。
- **区分「翻身」与「小动作」**:病人伸手、蹬腿都会引起压力/姿态变化,但不算有效翻身。要靠幅度阈值 + 持续时间 + 融合判断来滤掉。
- **长期佩戴的舒适与卫生**:若 IMU 要贴身,得考虑固定方式、皮肤刺激、可清洁——又回到护理场景「耐用可清洁」的老主题(见 CB4)。

### 动手:先做一个 IMU 体位识别

压力阵列涉及网格和多路复用,较复杂;IMU 这一路最容易先跑通,建议从它入手:

1. 把 **MPU-6050** 用 I²C 接到 ESP32(VCC/GND/SDA/SCL 四根线,回顾 BUS2)。
2. 先跑一个 I²C 扫描(见 BUS2 的扫描器)确认能看到它的地址(通常 0x68)。
3. 用现成库读加速度计三轴,算倾角:

```cpp
#include <Wire.h>
#include <MPU6050.h>
MPU6050 mpu;
void setup(){ Wire.begin(); Serial.begin(115200); mpu.initialize(); }
void loop(){
  int16_t ax, ay, az;
  mpu.getAcceleration(&ax, &ay, &az);
  // 静止时靠重力在各轴的分量判断朝向
  Serial.print(ax); Serial.print(' ');
  Serial.print(ay); Serial.print(' ');
  Serial.println(az);
  delay(200);
}
```

把传感器平放、立起、侧翻,观察哪个轴的读数接近 ±1g——你就亲眼看到「重力落在哪个轴」如何对应朝向(见前文原理)。据此设几个阈值,就能区分仰卧/左侧卧/右侧卧,这已经是一个能用的体位识别器。

想做压力阵列,再进一步:先用**单个 FSR** 接分压 + ADC 跑通「压力→数值」,再扩展成多点、加 CD74HC4067 多路复用逐行扫描(见前文)。同样是「先跑最小、再扩规模」。

### 常见问题排查(体位)

| 现象 | 可能原因 | 排查方向 |
|---|---|---|
| I²C 读不到 IMU | 没共地 / 没上拉 / 地址错 | 查 SDA/SCL、上拉电阻、先跑 I²C 扫描 |
| 朝向读数漂移 | 只用陀螺仪积分 | 融合加速度计(互补/卡尔曼滤波) |
| 静止时朝向还乱跳 | 运动加速度混入 / 振动 | 静止判据 + 滤波,稳固安装 |
| FSR 网格出现「鬼影」 | 扫描串扰 | 加二极管隔离或有源扫描 |
| 同样翻身有时测到有时测不到 | 阈值对不同人不适配 | 自适应校准、用相对变化 |

> ✦ **要点:** 翻身与体位靠**压力阵列 + IMU**双保险。压力垫(FSR / Velostat)用**分压**把压力转成电压,再经**多路复用器逐行扫描**成一张热力图;IMU(加速度计 + 陀螺仪,经姿态融合)经 **I²C** 给出稳定朝向。二者融合,才能可靠地做**翻身计时(防压疮)**、**离床报警(防跌倒)**与体位识别。单一传感器都有盲区,交叉验证是关键。
