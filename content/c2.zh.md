## 核心讲义

### 两根线,挂一串器件

UART 是点对点的:一对一。但如果你想在同两根线上同时挂 OLED 屏、时钟芯片、温度传感器好几个器件呢?**I²C(读作 I-squared-C 或 I-two-C)**就是为此而生:**只用两根线**,就能连接一主多从的一整条总线。

- **SDA(Serial Data):** 数据线
- **SCL(Serial Clock):** 时钟线,由主机产生,大家跟着它的节拍收发

因为有专门的时钟线,I²C 是**同步**的(对比 UART 的异步)。

### 每个从机都有地址

两根线上挂了好几个器件,主机怎么点名?靠**地址**。每个 I²C 从机出厂就有一个 **7 位地址**(0–127)。主机每次通信先广播「我要找 0x3C」,只有地址匹配的从机才应答,其余保持沉默。

| 器件 | 典型 7 位地址 |
|---|---|
| SSD1306 OLED | 0x3C |
| DS1307 时钟 | 0x68 |
| MPU6050 陀螺仪 | 0x68 / 0x69 |
| PCF8574 扩展 | 0x20–0x27 |

（注意地址会冲突,如 DS1307 和 MPU6050 都是 0x68,不能同时上同一条总线。)

### 一帧是怎么走的

上方交互图展示了一次寻址的时序。一次典型通信:

1. **START(起始条件):** SCL 为高时,SDA 由高变低——总线上「开始了」的信号。
2. **地址 + 读写位:** 主机发出 7 位地址,再跟 1 位 R/W(0=写,1=读)。
3. **ACK(应答):** 被点名的从机把 SDA 拉低一拍,表示「收到,是我」。没有 ACK 说明该地址没人。
4. **数据字节:** 一个或多个字节,每个字节后都有一次 ACK。
5. **STOP(停止条件):** SCL 为高时,SDA 由低变高——通信结束。

### 开漏与上拉电阻

I²C 的两根线都是**开漏(open-drain)**:器件只能把线**拉低**,不能主动拉高。所以总线必须各接一个**上拉电阻**(典型 **4.7 kΩ**)到 Vcc,平时把线维持在高电平。这也是为什么 I²C 允许多个器件共线而不打架——大家只往下拉,松手就自动回高。

### 用 Wire 库扫描与通信

一段实用的**地址扫描器**,帮你确认器件接对了、地址是多少:

```cpp
#include <Wire.h>
void setup() {
  Wire.begin();
  Serial.begin(9600);
  for (byte a = 1; a < 127; a++) {
    Wire.beginTransmission(a);
    if (Wire.endTransmission() == 0) {   // 收到 ACK
      Serial.print("Found 0x"); Serial.println(a, HEX);
    }
  }
}
void loop() {}
```

> ✦ **要点:** I²C 用两根线 + 地址,把多个器件挂到同一条总线。SDA 传数据、SCL 传时钟,主机点名、从机以 ACK 应答,靠开漏 + 上拉电阻共线。接不上时,先查上拉电阻和地址冲突。
