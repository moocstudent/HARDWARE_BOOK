## 核心讲义

### 最古老、最常用的串口

**UART(通用异步收发器)**是芯片之间最基础的通信方式,也是你上传 Arduino 程序、用 `Serial.println` 打印调试信息时背后所用的东西。它是**点对点**的:两个器件、两根数据线,一来一回。

- **TX(Transmit):** 发送
- **RX(Receive):** 接收

连接的铁律是**交叉**:一方的 TX 接另一方的 RX,反之亦然。再加一根**共地(GND)**作为公共参考。

```text
  A.TX ──────► B.RX
  A.RX ◄────── B.TX
  A.GND ────── B.GND
```

### 异步:没有时钟线

UART 里的 **A(Asynchronous,异步)**是关键:双方**不共享时钟线**。那接收方怎么知道每一位持续多久?——靠**事先约好一个速率**,即**波特率(baud rate)**。

波特率就是每秒传多少位。常见值:9600、115200……只要收发双方设成**同一个**波特率,接收方就能按这个节拍在正确时刻采样每一位。

> 位时间 = 1 / 波特率。9600 波特下,每位约 104 µs。

**波特率不一致是 UART 最常见的故障**:串口监视器里出现乱码,十有八九是波特率设错了。

### 一帧的结构

平时空闲线保持高电平。发送一个字节时,数据被打包成一**帧**:

1. **起始位(Start):** 1 位低电平,告诉接收方「要来了」,让它对齐节拍。
2. **数据位(Data):** 通常 8 位,**低位在前(LSB first)**。
3. **校验位(Parity):** 可选的 1 位,用于简单检错。
4. **停止位(Stop):** 1 位(或 2 位)高电平,标志一帧结束。

上方交互图可拖动要发送的字节,观察起始位、8 个数据位、停止位如何在线上排布。常说的「8N1」就是:8 数据位、无校验(No parity)、1 停止位。

### 用 Serial 通信

```cpp
void setup() {
  Serial.begin(9600);        // 设定波特率
}
void loop() {
  Serial.println("Hello");   // 发送一行
  if (Serial.available()) {  // 有收到数据吗?
    char c = Serial.read();  // 读一个字节
    Serial.print("got: ");
    Serial.println(c);
  }
  delay(500);
}
```

> ⚠ **要点:** UART 是异步点对点串口——没有时钟线,靠双方约定的**波特率**同步。记住三件事:**TX/RX 交叉**、**共地**、**波特率一致**。乱码几乎都是这三条里没做对。5V 与 3.3V 器件互连时,别忘了电平匹配。
