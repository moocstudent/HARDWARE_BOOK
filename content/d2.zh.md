## 核心讲义

### 悬空的输入脚会「乱跳」

一个设为 INPUT 的引脚是高阻态,像一根灵敏的天线。如果它没有连到任何确定电平——比如按钮没按下时,引脚一端悬空——它读到的值会随周围电磁干扰随机在 HIGH/LOW 之间跳变。这叫**悬空输入(floating input)**,是新手最常踩的坑之一:「我没碰它,按钮读数怎么自己在变?」

解决办法:给引脚一个**确定的默认电平**,让它在没有外力时稳稳停在 HIGH 或 LOW。这就是上拉/下拉电阻的工作。

### 上拉与下拉

- **上拉电阻(pull-up):** 一个电阻把引脚接到 Vcc。默认时引脚被「拉高」读 HIGH;按钮把引脚接到 GND,按下时读 LOW。→ **按下 = LOW**。
- **下拉电阻(pull-down):** 一个电阻把引脚接到 GND。默认读 LOW;按钮把引脚接到 Vcc,按下读 HIGH。→ **按下 = HIGH**。

```text
上拉:                    下拉:
 Vcc                      pin ─┬─ 按钮 ─ Vcc
  │                            │
  R (10kΩ)                     R (10kΩ)
  │                            │
 pin ─┬─ 按钮 ─ GND           GND
```

电阻取值通常 **10 kΩ** 左右:足够大以免默认状态下白白耗电,又足够小以稳定压制干扰。上方交互图可切换两种模式并按下按钮,观察 `digitalRead` 的结果。

### 内部上拉:省一个电阻

很多单片机在芯片内部就集成了上拉电阻,一行代码即可启用,连外部电阻都省了:

```cpp
pinMode(2, INPUT_PULLUP);   // 启用内部上拉,默认 HIGH
// 按钮另一端接 GND,按下读 LOW
```

这是读按钮最省事的标准写法(注意逻辑是「按下为 LOW」,别搞反)。

### 按钮抖动与消抖

机械按钮的金属触点在闭合/断开的一瞬间会**弹跳(bounce)**:短短几毫秒内快速通断好几次。单片机跑得飞快,会把一次按压误读成好几次。这叫**抖动**。

**软件消抖**——检测到变化后等一小段时间再确认:

```cpp
bool lastStable = HIGH;
unsigned long tChange = 0;

void loop() {
  bool raw = digitalRead(2);
  if (raw != lastStable && millis() - tChange > 20) { // 稳定 20ms
    tChange = millis();
    lastStable = raw;
    if (raw == LOW) { /* 确认一次按下 */ }
  }
}
```

**硬件消抖**——在引脚上并一颗小电容(和上拉电阻组成 RC,见 CF4),抖动被 RC 平滑掉。

> ⚠ **要点:** 输入脚永远要有确定的默认电平——外部上拉/下拉,或内部 `INPUT_PULLUP`。再加消抖,按钮才可靠。「读数乱跳」几乎总是这两件事没做。
