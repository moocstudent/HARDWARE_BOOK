## 核心讲义

### 最简单的两个模拟传感器

传感器五花八门,但入门最该先吃透这两个:**电位器**和**光敏电阻(LDR)**。它们都不「智能」、没有芯片、没有协议,原理纯粹是 CF3 学过的**分压**——把一个物理量变成一个可读的电压,交给 ADC(见 IO4)读进来。理解了它们,你就理解了一大类模拟传感器的骨架。

### 电位器:把「旋钮位置」变成电压

电位器是三脚元件:两端接 5V 和 GND,中间抽头(wiper)接到模拟脚。转动旋钮改变分压比例,抽头输出 0–5V 连续变化的电压:

```cpp
void setup() { Serial.begin(9600); }
void loop() {
  int raw = analogRead(A0);        // 0..1023
  Serial.println(raw);
  delay(100);
}
```

常配合 `map()` 把 0–1023 标定到你需要的范围,比如 LED 亮度 0–255、舵机角度 0–180:

```cpp
int bright = map(analogRead(A0), 0, 1023, 0, 255);
analogWrite(9, bright);
```

### 光敏电阻:把「光强」变成电压

光敏电阻(LDR / photoresistor)的阻值**随光照变化**:光越强,阻值越小(可从暗处的几百 kΩ 降到亮处的几 kΩ)。但 ADC 读不了电阻,所以要和一个**固定电阻**组成分压器:

```text
5V ── LDR ──┬── A0
            │
        固定电阻 R
            │
           GND
```

光变强 → LDR 阻值下降 → LDR 分到的电压变小 → A0 抽头电压升高。反之变暗则 A0 下降。于是「光强」被转成了「电压」,`analogRead(A0)` 就能读出明暗。

### 固定电阻怎么选

固定电阻的取值决定了在你关心的光照区间里输出摆动最明显。经验法则:**取与传感器在该环境下典型阻值相近的值**(常见 10 kΩ 起步)。取得太大或太小,曲线会挤在一端,分辨率变差。

### 标定:从读数到有意义的量

原始读数(0–1023)只是「相对明暗」。若想得到「照度」等真实物理量,需要**标定**:在几个已知条件下记录读数,拟合出换算关系。多数场合我们只需相对判断——比如「读数低于某阈值就开灯」,这时一个 `if` 就够了:

```cpp
if (analogRead(A0) < 300) digitalWrite(13, HIGH);  // 天黑,开灯
else                      digitalWrite(13, LOW);
```

> ✦ **要点:** 电位器和 LDR 都靠分压把物理量(位置、光强)变成电压,再用 ADC 读入。核心就三步:**搭分压 → analogRead → 用 map/阈值处理**。这正是模拟传感器的通用套路。
