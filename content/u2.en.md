## Core Notes

### An MCU with Wi-Fi and Bluetooth

The ESP32 is a **dual-core 32-bit** microcontroller whose headline feature is **built-in Wi-Fi and Bluetooth**. When a project needs connectivity — uploading data to the cloud, serving a control web page, joining an IoT platform — the ESP32 is almost the default. Its clock, memory and peripherals far exceed the Arduino Uno's.

### 3.3V logic: the single most important caution

The crucial difference from the Uno: **the ESP32 is 3.3V logic, not 5V**. Its pins are **not 5V-tolerant** — wiring a 5V sensor's output straight to an ESP32 input can damage the chip. When interfacing with 5V parts, use **level shifting** (a level-shifter module, or a simple divider — see CF3). Power is usually over USB (regulated on-board to 3.3V); don't feed 5V into a 3.3V pin.

### Pins can be "remapped"

The Uno's I²C, SPI and serial are fixed to specific pins. The ESP32 has an internal **GPIO matrix** that lets most peripheral functions **map flexibly to different pins** — `Wire.begin(SDA, SCL)` lets you name which two pins do I²C. But note some pins have special roles (boot strapping pins, input-only pins, pins wired to the Flash), so check your board's pinout before wiring.

### Wi-Fi and deep sleep

Connecting to Wi-Fi is a few lines:

```cpp
#include <WiFi.h>
void setup() {
  Serial.begin(115200);
  WiFi.begin("SSID", "password");
  while (WiFi.status() != WL_CONNECTED) { delay(300); Serial.print("."); }
  Serial.println(WiFi.localIP());
}
void loop() {}
```

Radio is power-hungry. Battery projects lean on **deep sleep**: the chip sleeps at tens of microamps most of the time, wakes on a timer or a pin, samples/reports, then sleeps again. That's the key to months of battery life.

> ⚠ **Key point:** ESP32 = a beefier dual-core MCU + wireless. Before using it, burn in: **3.3V logic, don't over-volt the pins, peripherals are remappable**. Networking on batteries relies on deep sleep. Its programming interface resembles the Uno's (Arduino framework); porting code mostly means adjusting pin voltages and peripheral init.
