## Core Notes

### Two wires, a string of devices

UART is point-to-point: one to one. But what if you want an OLED, a clock chip and a temperature sensor all hanging off the *same* two wires? **I²C** (say "I-squared-C" or "I-two-C") is built for exactly that: **just two wires** connect a whole one-master, many-slave bus.

- **SDA (Serial Data):** the data line
- **SCL (Serial Clock):** the clock line, generated by the master; everyone sends/receives to its beat

Because it has a dedicated clock line, I²C is **synchronous** (versus UART's asynchronous).

### Every slave has an address

With several devices on two wires, how does the master call one out? By **address**. Each I²C slave ships with a **7-bit address** (0–127). Every transaction begins with the master broadcasting "I want 0x3C"; only the slave whose address matches answers, the rest stay silent.

| Device | Typical 7-bit address |
|---|---|
| SSD1306 OLED | 0x3C |
| DS1307 clock | 0x68 |
| MPU6050 IMU | 0x68 / 0x69 |
| PCF8574 expander | 0x20–0x27 |

(Watch for clashes — the DS1307 and MPU6050 are both 0x68 and can't share one bus.)

### How a transaction runs

The interactive figure above shows the timing of one addressing phase. A typical transaction:

1. **START:** while SCL is HIGH, SDA goes HIGH→LOW — the "we're beginning" signal on the bus.
2. **Address + R/W bit:** the master sends the 7-bit address, then one R/W bit (0 = write, 1 = read).
3. **ACK:** the addressed slave pulls SDA LOW for one beat to say "got it, that's me". No ACK means nobody has that address.
4. **Data byte(s):** one or more bytes, each followed by an ACK.
5. **STOP:** while SCL is HIGH, SDA goes LOW→HIGH — the transaction ends.

### Open-drain and pull-up resistors

Both I²C lines are **open-drain**: a device can only pull a line **LOW**, never actively drive it HIGH. So the bus needs a **pull-up resistor** (typically **4.7 kΩ**) on each line to Vcc, holding it HIGH at rest. That's also why many devices can share the lines without a fight — everyone only pulls down, and releasing lets it float back up.

### Scanning and talking with the Wire library

A handy **address scanner** to confirm your device is wired right and find its address:

```cpp
#include <Wire.h>
void setup() {
  Wire.begin();
  Serial.begin(9600);
  for (byte a = 1; a < 127; a++) {
    Wire.beginTransmission(a);
    if (Wire.endTransmission() == 0) {   // got an ACK
      Serial.print("Found 0x"); Serial.println(a, HEX);
    }
  }
}
void loop() {}
```

> ✦ **Key point:** I²C uses two wires plus addresses to hang many devices on one bus. SDA carries data, SCL the clock; the master calls, the slave answers with an ACK, and open-drain plus pull-ups let them share the lines. When nothing responds, check the pull-ups and address clashes first.
