## Core Notes

### Four wires, fast and full-duplex

When you drive a colour display or read/write an SD card — **high-throughput** devices — I²C's two-wire speed may not be enough. **SPI (Serial Peripheral Interface)** trades **four wires** for high speed and **full-duplex** (send and receive at once), the fastest of the three buses.

- **SCK (Serial Clock):** clock, from the master
- **MOSI (Master Out, Slave In):** data master → slave
- **MISO (Master In, Slave Out):** data slave → master
- **CS / SS (Chip Select):** the master uses it to single out a slave

Like I²C it has a clock line, so SPI is **synchronous** too; but it uses **separate send and receive lines**, so it can transmit and receive simultaneously — and much faster.

### Shift registers: sending and receiving are one act

SPI's underlying model is **two shift registers connected head to tail** (one in the master, one in the slave). On each clock edge both shift one bit out and one bit in — so **transmit and receive happen together**. Even to just read, you must send a "dummy" byte to generate the clock and push the slave's data out:

```cpp
byte received = SPI.transfer(0x00);  // send 0x00, receive a byte back
```

### Chip-select: choosing among slaves with CS

I²C tells devices apart by address; SPI does it with a **chip-select line**: one CS per slave. To talk to a slave the master pulls its CS LOW (select), then HIGH when done (release). More slaves means more CS lines — that's SPI's cost: more devices, more pins.

```cpp
#include <SPI.h>
void setup() {
  pinMode(10, OUTPUT);          // CS
  SPI.begin();
}
void write_reg(byte reg, byte val) {
  digitalWrite(10, LOW);        // select the slave
  SPI.transfer(reg);
  SPI.transfer(val);
  digitalWrite(10, HIGH);       // release
}
```

### Clock polarity and phase (SPI modes)

SPI has four **modes** set by two parameters: **CPOL** (is the clock idle high or low) and **CPHA** (sample on the first or second clock edge). Master and slave must use the same mode or the data misaligns. The datasheet states which mode a device uses; configure it with `SPISettings` to match.

### The three buses at a glance

| | UART | I²C | SPI |
|---|---|---|---|
| Wires | 2 | 2 | 4+ |
| Clock | none (async) | yes | yes |
| Many slaves | no | by address | by chip-select |
| Speed | low–mid | mid | high |
| Good for | debug / point-to-point | several slow small parts | displays / SD cards |

> ✦ **Key point:** SPI = four wires, synchronous, full-duplex, managing many slaves with **chip-select**, and the fastest. The core intuition is "shift registers: sending is receiving". Align the **SPI mode (CPOL/CPHA)** before use. Needing more pins per device is its trade-off.
