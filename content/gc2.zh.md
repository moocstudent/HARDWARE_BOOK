## 核心讲义

### 掌机其实是一台跑 Linux 的小电脑

这些掌机没有神秘之处:它就是一台 ARM 架构的**小型 Linux 计算机**——和树莓派(见 MC 模块)是近亲。它没有内置固化系统盘,而是**从 SD 卡启动**。这带来一个关键的自由:换一张 SD 卡,就换了一整个操作系统。

### 从 SD 卡启动的引导链

按下电源后,发生的事和 PC 开机异曲同工,只是更精简:

```text
上电 → SoC 内部 BootROM → 从 SD 卡读取引导器(U-Boot)
     → U-Boot 加载 Linux 内核 + 设备树
     → 内核挂载根文件系统 → 启动模拟器前端(EmulationStation)
```

**设备树(Device Tree)**告诉内核「这台机器有哪些硬件、引脚怎么接」——同一份内核配上不同设备树,就能适配不同型号。理解这条链,你就明白为什么「刷错固件会黑屏」以及「换卡能救回来」。

### 原厂固件 vs 第三方固件

平价掌机自带的原厂系统往往**臃肿、有广告、更新停滞**。好在开源社区为这些常见 SoC 做了高质量的第三方固件,烧进 SD 卡即可替换:

| 固件 | 特点 |
|---|---|
| **ArkOS** | 基于 Debian,稳定、易上手,R36S 的常见默认之选 |
| **JELOS / ROCKNIX** | 基于 Linux + 只读系统,更「一体化」、更省心 |
| **AmberELEC** | 面向 RG351 系列,同类思路 |

刷第三方固件通常能带来:更好的性能调优、更全的模拟器、更干净的界面、活跃的更新与社区支持。

### 怎么刷:烧录镜像到 SD 卡

流程和给树莓派做系统卡一模一样:

1. 下载固件的镜像文件(`.img`);
2. 用 **balenaEtcher** 或 **Raspberry Pi Imager** 把镜像烧到 microSD 卡;
3. 插回掌机的**系统卡槽**,开机;首次启动会自动扩展分区、生成目录。

> ⚠ 刷机前先备份;认准你机型对应的固件版本(SoC 与设备树要匹配),刷错型号会黑屏——但因为系统在 SD 卡上,**重新烧一张卡即可救回**,不会变砖。

### 双卡槽:系统卡 vs 游戏卡

R36S 这类机器常有**两个 microSD 卡槽**,分工清晰:

- **卡槽 1(系统卡):** 放操作系统/固件。一般较小(如 16–32 GB)。
- **卡槽 2(游戏卡):** 放 ROM 与资源。可用大容量(如 128–256 GB)。

分开的好处:升级或更换系统时,游戏卡原封不动;备份也更简单。R36S 附带的 128 GB 卡通常就是预装游戏的「游戏卡」。

> ✦ **要点:** 掌机 = **从 SD 卡启动的 Linux 小电脑**,引导链是 BootROM→U-Boot→内核→前端。刷 **ArkOS/JELOS** 等第三方固件即可焕新;因为系统在卡上,刷坏也能重烧救回。善用**双卡槽**:系统与游戏分离。
