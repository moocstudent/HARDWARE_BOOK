## 核心讲义

### 掌机其实是一台跑 Linux 的小电脑

这些掌机没有任何神秘之处:它就是一台 ARM 架构的**小型 Linux 计算机**——和你在 MC 模块里认识的树莓派是近亲(同为 ARM SoC + Linux)。它和 Arduino 那种「单片机」不同:Arduino 跑的是你烧进去的一段固化程序,而掌机跑的是一个完整的操作系统,有内核、有文件系统、有多任务调度。

一个关键事实决定了它的可玩性:**它没有内置固化的系统盘,而是从 SD 卡启动。** 这意味着——换一张 SD 卡,就换了一整个操作系统。这个设计给了你巨大的自由,也是「刷机」这件事的物理基础。

### 从 SD 卡启动的引导链

按下电源后发生的事,和 PC 开机异曲同工,只是更精简。理解这条链,后面的一切(为什么能刷机、为什么刷错会黑屏、为什么能救回来)就都通了:

```text
上电
 → SoC 内部固化的 BootROM 运行
 → 从 SD 卡指定位置读取引导器(U-Boot)
 → U-Boot 加载 Linux 内核 + 设备树(Device Tree)
 → 内核初始化硬件、挂载根文件系统
 → 启动模拟器前端(EmulationStation)
 → 你看到选游戏的界面
```

几个关键角色:

- **BootROM**:固化在 SoC 内部、无法改动的一小段引导代码。上电后它第一个运行,唯一任务是「从哪里找到下一棒(引导器)」。正是它规定了「从 SD 卡启动」。
- **U-Boot**:一个功能强大的引导器,负责把内核和设备树从 SD 卡加载到内存、设好启动参数、然后把控制权交给内核。
- **设备树(Device Tree)**:一份描述「这台机器有哪些硬件、引脚怎么接、屏幕多大、按键在哪」的数据文件。**同一份 Linux 内核,配上不同的设备树,就能适配不同型号的掌机。** 这就是为什么固件必须和你的具体机型匹配——设备树错了,内核就找不到屏幕和按键,表现为黑屏或没反应。

理解了这条链,你就明白:**刷错固件顶多黑屏,而不会「变砖」——因为系统全在 SD 卡上,重新烧一张卡就能救回来。**

### 原厂固件 vs 第三方固件

平价掌机自带的原厂系统,往往**臃肿、有广告、更新停滞、体验粗糙**。好在开源社区为这些常见的 SoC(尤其瑞芯微系列)做了大量高质量的第三方固件,烧进 SD 卡即可整体替换:

| 固件 | 基于 | 特点 |
|---|---|---|
| **ArkOS** | Debian | 稳定、易上手、可定制,R36S 常见默认之选 |
| **JELOS / ROCKNIX** | 只读 Linux | 「一体化家电」式,系统只读更稳、更省心 |
| **AmberELEC** | 只读 Linux | 面向 Anbernic RG351 系列,同类思路 |
| **Batocera** | 只读 Linux | 跨平台通用,支持大量设备 |

刷第三方固件通常能带来:**更好的性能调优**(针对该芯片优化过的内核与内核选择)、**更全的模拟器与更新**、**更干净的界面**、以及**活跃的社区支持**(遇到问题有人答)。这也是 GC1 里说「看社区」是选机重要信号的原因——一台机器有活跃的第三方固件,基本等于「靠谱、好玩、坑已填平」。

「只读系统」型固件(JELOS 等)有个额外好处:系统分区只读,你怎么折腾游戏和配置都不会把系统本身弄坏,断电也不易损坏文件系统,非常适合掌机这种随时关机的设备。

### 怎么刷:把镜像烧到 SD 卡

流程和给树莓派做系统卡一模一样(回顾 MC 模块),非常简单:

1. **下载**你机型对应的固件**镜像文件**(通常是 `.img` 或压缩包)。**务必认准型号**——RK3326 的 R36S 要下 R36S/RG351 系列对应的版本,设备树要匹配。
2. 用 **balenaEtcher** 或 **Raspberry Pi Imager** 把镜像**烧录**到一张 microSD 卡。烧录会擦除卡上所有内容,先备份。
3. 把卡插回掌机的**系统卡槽**,开机。首次启动通常会**自动扩展分区**(把系统撑满整张卡)、生成 `roms`、`bios` 等目录。

> ⚠ **刷机三原则**:①先备份原卡(尤其是随机附带的、预装好游戏的卡);②认准与机型匹配的固件版本(SoC + 设备树);③刷坏别慌——因为系统在 SD 卡上,重烧一张即可救回,不会变砖。这也是掌机比手机、比 Arduino「更安全折腾」的地方。

### 双卡槽:系统卡 vs 游戏卡

R36S 这类机器常有**两个 microSD 卡槽**,这不是摆设,而是清晰的分工设计:

- **卡槽 1(系统卡 / TF1)**:放操作系统 / 固件。一般用较小的卡(如 16–32 GB)就够。
- **卡槽 2(游戏卡 / TF2)**:放 ROM 与资源(封面图等)。可用大容量(如 128–256 GB)。R36S 附带的那张 128 GB「预装上万游戏」的卡,通常就是插在这个槽的游戏卡。

**分开的好处**:
- **升级/换系统时,游戏卡原封不动**——你可以随便刷系统卡试不同固件,珍贵的游戏和存档不受影响。
- **备份更简单**——两类数据分离,各自备份。
- **容量灵活**——系统小、游戏大,各配合适的卡,省钱。

理解了双卡槽,你就知道「想换个固件试试」时该动哪张卡(系统卡),以及「加游戏」时该往哪张卡放(游戏卡,见 GC4 的目录布局)。

### SD 卡里到底有什么:分区布局

烧好的系统卡不是一个大文件,而是被分成几个**分区**,各司其职。理解这个布局,刷机、扩容、救砖时就不会一头雾水:

- **引导分区(boot)**:放 U-Boot、内核、设备树、启动脚本。BootROM 就是从这里找到下一棒的。通常较小、格式特殊。
- **系统分区(root/system)**:放操作系统本体(前端、RetroArch、库)。只读型固件(JELOS)会把它设为只读,保护系统不被写坏。
- **数据/游戏分区**:放你的 ROM、存档、配置。首次启动时,系统常会**自动把这个分区扩展**到填满整张卡的剩余空间(所以刚烧完是小的,开机后变大)。

烧录镜像其实就是把这一整套分区结构原样写进卡里;首次开机的「自动扩容」则是把最后那个数据分区撑大以利用整张卡。

### 更新固件 vs 全新重刷

固件用久了要更新,有两种方式,别搞混:

- **增量更新(在机内)**:很多固件带「在线更新 / OTA」功能,只更新系统文件,**保留你的游戏、存档、配置**。日常小版本升级用这个,省事。
- **全新重刷(重烧镜像)**:把整张系统卡重烧。会**清空系统卡**(游戏卡若分开则不受影响,见双卡槽)。换固件、或系统弄乱了要「恢复出厂」时用这个。

**重刷前务必备份存档**——存档通常在游戏卡或数据分区,但也有固件把它放在系统卡里,搞不清就先整卡备份一份(用 Win32DiskImager 之类做整卡镜像)。

### 黑屏了怎么办:救砖思路

新手最怕「刷完开机黑屏」。回到 GC 的核心认知——**系统在 SD 卡上,所以几乎不可能真变砖**。黑屏排查顺序:

1. **固件版本对不对?** 型号/设备树不匹配是黑屏第一原因。重新下载**你这台机器**对应的正确固件。
2. **烧录成功了吗?** 用 balenaEtcher 的校验功能确认写入无误;换一张卡试试(劣质卡是常见元凶)。
3. **卡插对槽了吗?** 系统卡要插系统槽(TF1)。
4. 以上都对还黑屏 → 查该机型社区的专门教程,常有针对性的按键组合或恢复镜像。

只要原厂卡还在(或你备份过),重烧一张就能回到能用的状态——这份「折腾不怕坏」的安全感,正是 Linux 掌机好玩的地方。

### 选固件的现实考量

同一台机器往往有多个第三方固件可选,怎么挑?

- **硬件支持**:优先选对**你这台机器**支持成熟的固件(尤其 Wi-Fi 适配器、屏幕、按键都被正确驱动)。查社区兼容列表。
- **只读 vs 可写**:只读型(JELOS)更稳、更防断电损坏,适合「拿来即玩」;可写型(ArkOS/Debian 系)更自由、方便你装软件跑脚本(见 GC4),适合想深度折腾的人。
- **界面与功能**:有的固件界面漂亮、封面刮削全;有的功能极客、可玩性高。按喜好试——反正换固件只是换张卡。
- **社区活跃度**:更新勤、问题有人答的固件,长期体验更好。

这也解释了为什么很多人会准备**多张系统卡**,各刷一个固件,随手一换就切换整个系统——这正是「从 SD 卡启动」这个设计给玩家的最大自由。

### 动手:第一次刷固件的完整流程

从零把一张系统卡刷好、开机的完整走一遍(以 ArkOS 到 R36S 为例,其它机型思路一致):

1. **备份原卡**:如果机器附带了预装卡,先用 Win32DiskImager / balenaEtcher 的「从卡读镜像」功能整卡备份一份,存好。这是你的后悔药。
2. **下对固件**:去该机型的官方/社区页面,下载**明确写明支持你这台机器**的固件镜像(R36S 通常用 RG351 系列的对应版本)。核对是给你这个屏、这个芯片的版本。
3. **烧录**:把一张够快的正品 microSD 插电脑,用 **balenaEtcher** 选镜像、选卡、烧录,等它校验通过。
4. **插卡开机**:把卡插进**系统卡槽(TF1)**,开机。首次启动它会自动扩容、初始化,可能重启一两次,耐心等。
5. **首启设置**:进系统后设语言、时区;若有 Wi-Fi 适配器,连网;确认按键、屏幕、声音都正常。
6. **放游戏**:把 ROM 放进游戏卡(或同卡)的 `/roms/<主机>/` 目录,回前端「更新游戏列表」(详见 GC4)。

全程最容易错的两步是「下错固件版本」和「用了劣质/山寨卡」——记住这两点,基本一次成功。

### 常见问题排查(固件)

| 现象 | 可能原因 | 排查方向 |
|---|---|---|
| 开机黑屏 / 无反应 | 固件与机型不符(设备树错) | 重下你机型专用固件,别用别的型号的 |
| 反复重启进不去 | 卡质量差 / 烧录未校验 | 换正品快卡,重烧并校验 |
| 能进但没 Wi-Fi | 适配器芯片内核不支持 | 换受支持的 Wi-Fi 芯片(见 GC4) |
| 游戏卡空间变小 | 首启没自动扩容 / 分区没撑满 | 用工具扩展分区,或重刷 |
| 刷完想回原样 | — | 用第 1 步的备份镜像重烧回去 |

### 进阶:多系统、备份与「不怕坏」的心态

理解了「系统在 SD 卡上」,就能玩出更多花样,也能建立一套稳妥的折腾习惯:

- **多张系统卡,想切就切**:准备几张系统卡,分别刷 ArkOS、JELOS、Batocera,开机换卡就换了整个系统——评测对比、按心情选界面,互不影响。这是「从 SD 启动」给玩家的独特自由。
- **整卡镜像备份**:折腾前用 Win32DiskImager「读镜像」把整卡存成一个 `.img` 文件。哪天弄坏了,一键写回,秒回到能用状态。这比手机、比 Arduino 都安全——**你几乎不可能真把它弄砖**。
- **存档单独备份**:系统可以重刷,存档丢了没了(见 GC4)。养成定期把存档拷出来的习惯。
- **先在便宜卡上试**:重要操作先在一张不重要的卡上试通,再动你的主力卡。

这套「备份 + 可重刷」的安全网,正是 Linux 掌机适合学习和折腾的根本原因:**代价极低的试错**。你可以大胆改、大胆刷,反正坏了能救——这种「不怕坏」的心态,恰恰是学硬件、学 Linux 最宝贵的环境。

### 常见误区澄清

| 说法 | 澄清 |
|---|---|
| 「刷机会把机器刷坏」 | 系统在卡上,刷坏重烧即可,几乎不会变砖 |
| 「第三方固件不安全」 | 主流开源固件成熟活跃,常比原厂更好 |
| 「双卡槽一个是备用」 | 不是备用,是系统卡 + 游戏卡分工 |
| 「换固件游戏就没了」 | 游戏在游戏卡上,换系统卡不影响 |
| 「黑屏就是变砖了」 | 多半是固件不匹配,重下正确版本即可 |
