## Core Notes

### The two simplest analog sensors

Sensors come in endless variety, but the two to master first are the **potentiometer** and the **photoresistor (LDR)**. Neither is "smart" — no chip, no protocol — their principle is pure **voltage division** from CF3: turn a physical quantity into a readable voltage and hand it to the ADC (see IO4). Understand these and you understand the skeleton of a whole class of analog sensors.

### Potentiometer: knob position → voltage

A potentiometer is a three-terminal part: the ends connect to 5V and GND, the wiper to an analog pin. Turning the knob changes the divider ratio, so the wiper outputs a voltage that varies continuously from 0–5V:

```cpp
void setup() { Serial.begin(9600); }
void loop() {
  int raw = analogRead(A0);        // 0..1023
  Serial.println(raw);
  delay(100);
}
```

It's often paired with `map()` to scale 0–1023 to whatever range you need — LED brightness 0–255, servo angle 0–180:

```cpp
int bright = map(analogRead(A0), 0, 1023, 0, 255);
analogWrite(9, bright);
```

### Photoresistor: light → voltage

A photoresistor (LDR) changes **resistance with light**: brighter means lower resistance (from hundreds of kΩ in the dark down to a few kΩ in bright light). But the ADC can't read resistance, so it forms a divider with a **fixed resistor**:

```text
5V ── LDR ──┬── A0
            │
       fixed resistor R
            │
           GND
```

More light → LDR resistance drops → less voltage across the LDR → the A0 tap voltage rises. Darker and A0 falls. So "light level" becomes "voltage", and `analogRead(A0)` reads bright vs. dark.

### Choosing the fixed resistor

The fixed resistor's value decides where the output swings most over the light range you care about. Rule of thumb: **pick it close to the sensor's typical resistance in that environment** (10 kΩ is a common starting point). Too large or too small and the curve bunches at one end, hurting resolution.

### Calibration: from a reading to a meaningful quantity

The raw reading (0–1023) is only "relative brightness". To get a true physical quantity like lux, you **calibrate**: record readings under a few known conditions and fit a mapping. Most of the time you only need a relative decision — "turn a light on below some threshold" — where a single `if` suffices:

```cpp
if (analogRead(A0) < 300) digitalWrite(13, HIGH);  // dark, light on
else                      digitalWrite(13, LOW);
```

> ✦ **Key point:** the potentiometer and LDR both use a divider to turn a physical quantity (position, light) into a voltage, then read it with the ADC. Three steps: **build the divider → analogRead → process with map/threshold**. That's the universal pattern for analog sensors.
