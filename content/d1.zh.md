## 核心讲义

### 数字世界只有两个值

模拟世界里电压是连续的,但数字芯片只认两个状态:**HIGH(高,逻辑 1)**和**LOW(低,逻辑 0)**。在 5V 系统里,接近 5V 算 HIGH,接近 0V 算 LOW;3.3V 系统里则以 3.3V 为高。

芯片用**阈值**来判断:高于某电压算 1,低于某电压算 0,中间有一段「模糊地带」应尽量避免让输入停留。这也是为什么信号要「干脆利落」地在高低之间跳变。

### pinMode:先决定引脚是「说」还是「听」

每个 GPIO(通用输入输出)引脚,用之前要先声明方向:

```cpp
pinMode(pin, OUTPUT);       // 输出:引脚主动驱动高/低
pinMode(pin, INPUT);        // 输入:引脚被动读取外部电平
pinMode(pin, INPUT_PULLUP); // 输入 + 内部上拉(见 IO2)
```

- **OUTPUT(输出):** 引脚变成一个能主动输出 5V 或 0V 的「小电源」,可以点灯、驱动蜂鸣器等(但电流有限,通常单脚 ≤ 20 mA)。
- **INPUT(输入):** 引脚变成「高阻态」——几乎不吸取电流,安静地感知外部电压,用来读按钮、传感器。

### digitalWrite:让引脚说话

```cpp
digitalWrite(13, HIGH);  // 输出约 5V
digitalWrite(13, LOW);   // 输出 0V
```

配合 CF2 学过的「引脚 → 电阻 → LED → GND」,这就是点灯。经典 Blink:

```cpp
void setup() { pinMode(13, OUTPUT); }
void loop() {
  digitalWrite(13, HIGH); delay(500);
  digitalWrite(13, LOW);  delay(500);
}
```

### digitalRead:让引脚倾听

```cpp
int state = digitalRead(2);   // 返回 HIGH(1) 或 LOW(0)
if (state == HIGH) { /* ... */ }
```

读按钮的最小程序(配合内部上拉,按下为 LOW):

```cpp
void setup() {
  pinMode(2, INPUT_PULLUP);
  pinMode(13, OUTPUT);
}
void loop() {
  bool pressed = (digitalRead(2) == LOW);
  digitalWrite(13, pressed ? HIGH : LOW);
}
```

### 灌电流与拉电流

输出 HIGH 时,电流从引脚**流出**去驱动负载,叫**拉电流(source)**;输出 LOW 时,电流从负载**流入**引脚到地,叫**灌电流(sink)**。两者都有上限(ATmega328P 单脚约 20–40 mA,总和还有限制)。驱动电机、继电器等大电流负载时,绝不能直接接引脚,要用三极管/MOS 管/驱动板(见 AD2)。

> ✦ **要点:** 数字 I/O 的全部秘密,就是 `pinMode` 定方向、`digitalWrite` 输出、`digitalRead` 读入。先把这三件事和「HIGH/LOW 到底是多少伏」对上号,后面所有外设都是它们的组合。
