## Core Notes

### The digital world has only two values

In the analog world voltage is continuous, but a digital chip recognises just two states: **HIGH (logic 1)** and **LOW (logic 0)**. In a 5V system, near 5V counts as HIGH and near 0V as LOW; in a 3.3V system HIGH is 3.3V.

The chip decides with a **threshold**: above one voltage it's a 1, below another it's a 0, with a "no man's land" in between that an input should not linger in. That's why signals should swing crisply between high and low.

### pinMode: first decide whether a pin speaks or listens

Before you use a GPIO (general-purpose I/O) pin, declare its direction:

```cpp
pinMode(pin, OUTPUT);       // output: the pin actively drives high/low
pinMode(pin, INPUT);        // input: the pin passively reads an external level
pinMode(pin, INPUT_PULLUP); // input + internal pull-up (see IO2)
```

- **OUTPUT:** the pin becomes a little source that actively drives 5V or 0V — enough to light an LED, sound a buzzer, etc. (but with limited current, usually ≤ 20 mA per pin).
- **INPUT:** the pin becomes high-impedance — it draws almost no current and quietly senses the external voltage, for reading buttons and sensors.

### digitalWrite: make the pin speak

```cpp
digitalWrite(13, HIGH);  // outputs ~5V
digitalWrite(13, LOW);   // outputs 0V
```

Combined with the "pin → resistor → LED → GND" from CF2, this lights an LED. The classic Blink:

```cpp
void setup() { pinMode(13, OUTPUT); }
void loop() {
  digitalWrite(13, HIGH); delay(500);
  digitalWrite(13, LOW);  delay(500);
}
```

### digitalRead: make the pin listen

```cpp
int state = digitalRead(2);   // returns HIGH(1) or LOW(0)
if (state == HIGH) { /* ... */ }
```

A minimal read-button sketch (with the internal pull-up, pressed reads LOW):

```cpp
void setup() {
  pinMode(2, INPUT_PULLUP);
  pinMode(13, OUTPUT);
}
void loop() {
  bool pressed = (digitalRead(2) == LOW);
  digitalWrite(13, pressed ? HIGH : LOW);
}
```

### Sourcing and sinking current

Driving HIGH, current flows *out* of the pin into the load — **sourcing**. Driving LOW, current flows *from* the load into the pin to ground — **sinking**. Both have limits (an ATmega328P pin handles ~20–40 mA, with a cap on the total too). For high-current loads like motors or relays, never wire them straight to a pin — use a transistor/MOSFET/driver board (see AD2).

> ✦ **Key point:** the whole secret of digital I/O is `pinMode` for direction, `digitalWrite` to output, `digitalRead` to read. Pin those three to "how many volts HIGH/LOW actually are" and every later peripheral is just a combination of them.
