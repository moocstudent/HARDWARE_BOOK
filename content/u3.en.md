## Core Notes

### RP2040: Raspberry Pi's own dual-core chip

The Raspberry Pi Pico is built on the **RP2040** — a **dual-core ARM Cortex-M0+** at 133 MHz, 3.3V logic. It's cheap, well-supported, and has one trick nobody else offers: **PIO**.

### PIO: "build" peripherals from state machines

An ordinary MCU's peripherals (UART, SPI, etc.) are fixed in silicon — you get however many the chip shipped with. The RP2040's **PIO (Programmable I/O)** gives you 8 tiny **state machines** you can program with a handful of instructions to **generate or sample arbitrary timing on the pins with precision**.

That means: if you need an interface the chip doesn't have (say the demanding timing of a WS2812 LED strip, or several extra serial ports), you don't tie up the CPU in a busy-wait — hand it to a PIO to run exactly. It's the RP2040's most distinctive, most worthwhile idea.

### Two development languages

The Pico supports two paths; pick by taste:

- **C/C++ (official SDK):** best performance, closest to the metal; the Arduino framework works too.
- **MicroPython:** a slim Python on the microcontroller — interactive, very quick to start, great for teaching and rapid prototyping.

Blinking in MicroPython:

```python
from machine import Pin
from time import sleep
led = Pin(25, Pin.OUT)     # onboard LED
while True:
    led.toggle()
    sleep(0.5)
```

### Pins and power essentials

The Pico has 26 usable GPIO, 3 ADC inputs (12-bit), 3.3V logic. Power over USB's `VBUS` (5V), with on-board regulation to `3V3`. Like the ESP32, **its pins are not 5V-tolerant** — level-shift when connecting 5V parts.

> ✦ **Key point:** the RP2040 is a well-balanced dual-core M0+ whose highlight is **PIO** — offloading timing-sensitive peripherals from the CPU to programmable state machines. Develop in C/C++ or MicroPython. Remember the 3.3V logic.
