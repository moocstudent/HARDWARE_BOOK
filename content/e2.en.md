## Core Notes

### An LED is not a resistor

A light-emitting diode (LED) is a **diode**: current flows only from the positive side (anode, long leg) to the negative side (cathode, short leg / flat edge), and barely at all in reverse. More importantly, its voltage–current relationship is **not** a straight line but a threshold:

- Below a certain voltage it barely conducts and doesn't light;
- Once past that threshold — the **forward voltage, Vf** — current rises **steeply**, while the voltage across it stays pinned near Vf.

| LED colour | Typical Vf | Typical current |
|---|---|---|
| Red | ~1.8–2.0 V | 5–20 mA |
| Yellow / orange | ~2.0–2.2 V | 5–20 mA |
| Green | ~2.0–3.0 V | 5–20 mA |
| Blue / white | ~3.0–3.4 V | 5–20 mA |

Because current is so sensitive to voltage, **you can't control the current directly**. Wire 5V straight to a red LED with Vf=2V and the math asks for infinite current — in practice, an instant burnout.

### The current-limiting resistor: one resistor saves one LED

The fix is a **series resistor** that soaks up the "extra" voltage and holds the current in a safe range. Part of the supply Vs drops across the LED (Vf); the rest drops across the resistor:

> Voltage across the resistor = Vs − Vf  
> so **R = (Vs − Vf) / I**

Example: 5V supply, red LED (Vf=2V), target 10 mA:

```text
R = (5 − 2) / 0.010 = 300 Ω
```

No exact 300Ω on hand? Pick the next larger standard value (e.g. 330Ω) — slightly less current, slightly dimmer, but **absolutely safe**. When in doubt, go bigger.

### Mind the polarity

- **Long leg = anode (+)**, toward the supply / MCU pin;
- **Short leg / flat side of the rim = cathode (−)**, toward ground (GND).

Reversed, the LED simply won't light (ordinary LEDs usually survive it, having enough reverse breakdown margin).

### Lighting it from an Arduino

Wire "pin → resistor → LED → GND" and the code is the classic Blink:

```cpp
void setup() {
  pinMode(13, OUTPUT);      // pin 13 as output
}
void loop() {
  digitalWrite(13, HIGH);   // pin drives ~5V, LED on
  delay(500);
  digitalWrite(13, LOW);    // pin drives 0V, LED off
  delay(500);
}
```

> ⚠ **Never skip the resistor.** "Let me just touch it to see if it lights" is the number-one way beginners kill LEDs. Even for a quick test, put a 220Ω or 330Ω in series. In the interactive figure above, lower the resistance and watch the current shoot into the red "over-current" zone.
