## 核心讲义

### 尿床检测:看似简单,细节最多

「床湿了没有」听上去是四个子系统里最简单的一路——不就是「检测有没有水」吗?但恰恰是它,最贴近真实护理现场,坑也最多。一个能用的尿床监测,要同时满足几个互相矛盾的要求:

- **足够灵敏**:尿失禁发生后要尽快发现,否则皮肤长时间浸渍会加速感染与压疮。
- **不误报**:出汗、翻身压出的体液、不小心洒的水,都不该触发报警。护理人员被误报折腾几次就会关掉报警——那这套系统就白做了。
- **耐腐蚀**:尿液是含盐的电解质,对金属电极腐蚀性强,传感器要能长期泡在这种环境里。
- **可清洁**:床垫要反复擦拭消毒,传感器不能怕水、怕擦。

这些要求的张力,正是这一章要处理的工程核心:**如何既灵敏又不误报,既能测水又耐得住水。**

### 湿度电极:水让两极导通

最直接的原理是**电导法**:两条靠得很近、互相交错的导电电极(称为**叉指电极 / interdigitated electrode**,因为形状像交叉的手指)。干燥时,两极之间是绝缘的,电阻极大;一旦有尿液(含电解质的导电液体)桥接在两极之间,离子导电让电阻骤降。把这对电极接成分压(见 CF3)或用专门的电容/电阻测量电路读出,就能发现「湿了」。

叉指结构的好处是:用很小的面积就能做出很长的「相邻边界」,任何一处被液体桥接都能被检测到,灵敏度高、响应快。

### 电容式 vs 电阻式:一场关于耐腐蚀的取舍

同样是测潮湿,有两种测量方式,取舍关键在**耐腐蚀**:

- **电阻式(电导式)**:直接测两极间电阻。电路最简单、成本最低,但电极必须**直接接触**电解液才能导电——而长期泡在含盐尿液里,金属电极会**电解腐蚀**、氧化,读数漂移甚至失效。
- **电容式**:测两极间介质的**介电常数变化**(水的介电常数远高于空气)。电极可以覆盖一层薄薄的绝缘介质、**不直接接触液体**,靠电场感知附近的水分。因此它**更耐腐蚀、更耐用**,寿命长得多。

> 结论:**电容式比电阻式更耐腐蚀**,产品化更倾向电容式。原型阶段可用现成的「土壤湿度式电容传感器」模块起步——它本就是为长期埋在潮湿土里设计的,思路与需求高度吻合。

### 用 SHT40 交叉确认「真湿」

只靠一路湿度电极,误报几乎不可避免:电极碰到一点汗、一点凝露就可能触发。解决之道,又回到本项目反复强调的**传感器融合 / 交叉验证**:加一颗 **SHT31 / SHT40 温湿度传感器**(**I²C** 接口,见 BUS2)做旁证。

真正的尿湿事件有几个可交叉印证的特征:

- **电极导通**(有导电液体桥接);
- **局部相对湿度骤升**(液体蒸发使传感器附近湿度快速上升);
- **温度接近体温**(刚排出的尿液是温的,约 35–37 °C,而洒的凉水、凝露温度偏低)。

把这三条用逻辑「与」起来判定,才算一次真事件:

```text
真尿湿  ≈  电极导通  AND  局部湿度快速↑  AND  温度≈体温
误触发  ≈  仅满足其一(出汗、洒凉水、空调凝露)
```

这样一来,汗液(温度对但湿度上升慢、面积小)、洒的凉水(温度不对)都能被排除,误报率大幅下降。SHT40 精度高、功耗低、I²C 接口简单,是这类交叉确认的理想选择。

### 耐用与可清洁:防水处理

传感器要长期工作在潮湿、需反复清洁擦拭的环境,如果不做防护,几周就会腐蚀失效。必须做**防护处理**:

- **三防漆(conformal coating)**:在电路板和非感应区涂一层保护漆,隔绝湿气,防止铜箔和焊点被腐蚀。这是电子在潮湿/户外环境里最常用、最便宜的保护手段。
- **硅胶密封**:把接线、电路的接缝用硅胶封住,防止液体渗入。
- **可擦拭隔膜**:在感应电极上覆一层薄膜或防水透气膜——它能把潮湿「传导」给电容式传感器感知,又形成一个可以直接擦拭、消毒的表面,把电极和病人、和清洁剂隔开。

这些看似琐碎的细节,恰恰决定了这套系统能不能在真实病房里「用得住」。一个实验室里跑得漂亮、装到床上两周就腐蚀失灵的传感器,是没有价值的。**在医疗与护理场景,可靠性、耐用性、可清洁性和精度同等重要——甚至更重要。**

### 现实中的坑

- **报警策略**:检测到湿之后,是立刻响,还是确认几秒再响?一般加一个短延时确认,避免瞬时误触。报警也要分级:是提示还是紧急。
- **定位**:大床上尿湿可能只发生在某一区域。可用多个电极分区,既能定位,也降低单点失效风险。
- **基线漂移**:环境湿度会随天气变化,电容读数的「干燥基线」不是固定值。要用相对变化(突然升高)而非绝对阈值来判定。
- **清洁与更换**:隔膜和电极是易损件,设计上要方便更换。

### 电容式到底怎么「隔着介质」测到水

电容式为什么不用接触液体也能测到潮湿?回顾 CF4 学过的电容:两块极板之间填充的**介质**,其**介电常数**决定了电容值。而水的介电常数(约 80)远高于空气(约 1)或干燥织物。

所以叉指电极即使覆盖一层薄薄的绝缘膜、不直接碰到尿液,只要附近的织物从「干」变「湿」,电极之间那片区域的等效介电常数就大幅上升 → 电容显著变大。测量电路只要检测这个电容变化(通常把电容接进一个振荡电路,测频率变化;或用专门的电容测量芯片),就知道「湿了」。**电场能穿透薄绝缘层去感知外面的水,而电流不必真的流过液体**——这正是电容式耐腐蚀的根本原因:金属电极被绝缘层保护着,不与电解液直接反应。

### 电阻式的简单实现

如果只是做原型、先跑通,电阻式其实非常简单,一颗电阻加一个 ADC 引脚就够(回顾 CF3 分压):

```text
Vcc ── 叉指电极 ──┬── ADC 引脚
                  │
              固定电阻 R(如 100kΩ)
                  │
                 GND
```

干燥时电极电阻极大,抽头几乎被拉到 GND,ADC 读到接近 0;一旦尿液桥接,电极电阻骤降,抽头电压升高,ADC 读数明显上跳。设一个阈值即可判「湿」。它的缺点前面说过——电极长期泡电解液会腐蚀,所以只适合原型或可频繁更换电极的场景。理解这个最简实现,能帮你看清电容式「贵在哪、好在哪」。

### 误报的敌人:出汗、洒水、凝露

这一章反复强调「不误报」,因为现实里干扰源太多了。把它们和「真尿湿」逐一对比,就明白为什么单一传感器不够、非得交叉验证:

| 事件 | 电极导通? | 局部湿度? | 温度? | 面积/速度 |
|---|---|---|---|---|
| **真尿湿** | 是(强) | 快速骤升 | ≈体温(温) | 大面积、快速 |
| 出汗 | 弱/慢 | 缓慢小升 | ≈体温 | 分散、缓慢 |
| 洒凉水 | 是 | 升 | 偏凉 | 视情况 |
| 空调凝露 | 弱 | 环境性 | 偏凉 | 缓慢 |

可见没有任何**单一**特征能干净地区分它们:光看导通,汗和水都会误触;光看温度,汗也是体温。但把「强导通 + 湿度快速骤升 + 温度接近体温 + 短时间大面积」几个条件**与**起来,就能把真尿湿从一堆干扰里挑出来。这张表就是 CB4 那条判定逻辑的由来,也是本项目「多传感器交叉验证」哲学最具体的一次落地。

### 分区与定位

一张成人护理床面积不小,尿湿往往只发生在局部。用**多个电极分区布置**(如按床面分成 4–6 个区),不仅能报「湿了」,还能报「哪块湿了」,方便护理人员快速处理;同时降低单点电极失效导致整床失守的风险,也让某一区被污染/损坏时其余区仍能工作——这又呼应了 CB1 讲的**冗余**与**优雅降级**。

### 报警时机与「确认窗口」

检测到湿之后,不该立刻尖叫。合理做法是设一个短的**确认窗口**:信号超过阈值并持续 N 秒(排除瞬时误触、电气毛刺)才判定为真事件、再报警。报警本身也要分级——是「提醒护理人员方便时处理」还是「紧急」。还要考虑**去重**:一次尿湿事件在处理前会持续触发,系统应报一次、然后进入「已知湿、等待处理」状态,而不是每几秒响一遍把人逼疯。这些看似琐碎的报警逻辑,恰恰决定护理人员愿不愿意信任并保留这个功能(参见 CB1 对报警系统的讨论)。

### 动手:先做一个「湿了报警」

用最简单的电阻式起步验证原理,再加 SHT40 做交叉确认:

1. **做电极**:两条平行的裸铜线/铜箔,靠近但不相碰,固定在一小块布上,就是最简易的湿度电极。
2. **接分压**:电极一端接 Vcc,另一端接「ADC 引脚 + 一颗固定电阻到 GND」(见前文电阻式接法)。
3. **读值**:

```cpp
void setup(){ Serial.begin(115200); }
void loop(){
  int v = analogRead(A0);
  Serial.println(v);            // 干燥时接近 0,滴几滴盐水会明显上跳
  delay(200);
}
```

用滴管滴几滴盐水(模拟尿液)到电极上,看读数跳变,设个阈值就能「报警」。这一步就跑通了检测。

4. **加交叉确认**:把 **SHT40** 用 I²C 接上(回顾 BUS2),读温湿度。把判定改成「电极超阈值 **且** 湿度上升 **且** 温度接近体温」才报警(见前文融合逻辑),对着「滴凉水」和「哈气(模拟出汗)」测试,验证误报被排除。

体会一下:光有电极,滴凉水也会误报;加了温湿度交叉确认后,只有「温 + 湿 + 导通」同时满足才报——这就是传感器融合把误报压下去的直观演示。

### 常见问题排查(湿度)

| 现象 | 可能原因 | 排查方向 |
|---|---|---|
| 干燥时读数就很高/乱跳 | 输入悬空 / 电极受潮残留 | 检查固定电阻分压、擦干电极 |
| 滴水没反应 | 电极间距太大 / 用了纯净水(不导电) | 缩小间距、用含盐水测试 |
| 用几天后灵敏度下降 | 电极腐蚀 | 改电容式、加绝缘膜、可更换设计 |
| 出汗/凝露误报 | 只用了电极单一判据 | 加 SHT40 温湿度交叉确认 |
| 一次尿湿反复响 | 没做事件去重 | 报一次后进入「已知湿」状态 |

### 进阶:融入护理流程

一个尿床传感器要真正减轻护理负担,不能只是「响一声」,还要融入完整的护理闭环:

- **分级通知**:床旁提示灯 + 护士站/手机推送,让不在房间的护理人员也能及时知道。
- **响应确认**:护理人员处理后能一键「已处理」,系统复位、记录处理时长。
- **趋势记录**:统计每晚发生次数与时间分布,帮助评估病情、调整护理计划(如夜间定时提醒如厕)。
- **与其它子系统联动**:尿湿往往伴随体动,可与 CB3 的体位数据交叉,减少误报、提供更完整的画面。

至此,「测到水」这个最朴素的功能,才升级成一个真正有护理价值的系统——这也呼应本项目反复强调的:**传感器只是起点,把数据变成可靠、可用、融入流程的判断与行动,才是难点所在。**
