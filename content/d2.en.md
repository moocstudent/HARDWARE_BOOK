## Core Notes

### A floating input reads garbage

A pin set to INPUT is high-impedance, like a sensitive antenna. If it isn't tied to a defined level — e.g. a button that leaves the pin dangling when not pressed — the value it reads flips randomly between HIGH and LOW with the surrounding electrical noise. This is a **floating input**, one of the most common beginner traps: "I didn't touch it, why is the button reading changing on its own?"

The fix: give the pin a **defined default level** so it sits firmly at HIGH or LOW when nothing else acts on it. That's the job of a pull-up or pull-down resistor.

### Pull-up vs pull-down

- **Pull-up:** a resistor ties the pin to Vcc. By default the pin is pulled HIGH; the button connects the pin to GND, so pressing reads LOW. → **pressed = LOW**.
- **Pull-down:** a resistor ties the pin to GND. By default it reads LOW; the button connects the pin to Vcc, so pressing reads HIGH. → **pressed = HIGH**.

```text
Pull-up:                  Pull-down:
 Vcc                      pin ─┬─ button ─ Vcc
  │                            │
  R (10kΩ)                     R (10kΩ)
  │                            │
 pin ─┬─ button ─ GND         GND
```

The resistor is typically around **10 kΩ**: large enough not to waste current in the default state, small enough to firmly suppress noise. In the interactive figure you can switch modes and hold the button to watch what `digitalRead` returns.

### The internal pull-up: save a resistor

Many MCUs build a pull-up resistor into the chip; enable it in one line and skip the external part entirely:

```cpp
pinMode(2, INPUT_PULLUP);   // enable internal pull-up, default HIGH
// wire the button's other side to GND; pressing reads LOW
```

This is the standard, least-fuss way to read a button (just remember the logic is "pressed = LOW", don't invert it).

### Button bounce and debouncing

A mechanical button's metal contacts **bounce** the instant they close or open: they make and break several times within a few milliseconds. The MCU runs so fast it reads one press as several. This is **bounce**.

**Software debounce** — after a change, wait a moment to confirm it:

```cpp
bool lastStable = HIGH;
unsigned long tChange = 0;

void loop() {
  bool raw = digitalRead(2);
  if (raw != lastStable && millis() - tChange > 20) { // stable 20 ms
    tChange = millis();
    lastStable = raw;
    if (raw == LOW) { /* one confirmed press */ }
  }
}
```

**Hardware debounce** — a small capacitor across the pin (forming an RC with the pull-up, see CF4) smooths the bounce away.

> ⚠ **Key point:** an input pin must always have a defined default level — an external pull-up/pull-down, or the internal `INPUT_PULLUP`. Add debouncing and the button becomes reliable. "Readings jump around" is nearly always one of these two things missing.
