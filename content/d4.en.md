## Core Notes

### The world is continuous, the chip is discrete

Temperature, light, knob position… real-world quantities are **continuous**. But inside an MCU there is only 0 and 1. For the chip to "read" an analog voltage it needs an **ADC (Analog-to-Digital Converter)** that **quantizes** a continuous voltage into an integer.

**Quantization** slices a voltage range into many equal steps, then reports which step the input landed on. More steps (more bits) means a finer reconstruction.

### Resolution: 10-bit = 0–1023

An ADC's **bit depth** sets the number of steps. The Arduino Uno's ADC is **10-bit**, i.e. 2¹⁰ = **1024** levels, reading **0–1023**:

- reading 0 ↔ 0 V
- reading 1023 ↔ the reference voltage (5 V by default on the Uno)
- each step ≈ 5V / 1024 ≈ 4.9 mV

### analogRead and converting to volts

`analogRead(pin)` returns the quantized result in one step:

```cpp
int raw = analogRead(A0);          // 0..1023
float volts = raw * 5.0 / 1023.0;  // convert back to volts
```

In the interactive figure, drag the input voltage to see the staircase transfer curve and how the reading steps up as the voltage rises.

With the divider from CF3, reading a potentiometer:

```cpp
void setup() { Serial.begin(9600); }
void loop() {
  int raw = analogRead(A0);
  float v = raw * 5.0 / 1023.0;
  Serial.print(raw); Serial.print("  "); Serial.println(v);
  delay(200);
}
```

### Reference voltage and sample rate

- **Reference voltage Vref:** the voltage that maps to reading 1023. It defaults to the 5V supply, but a steadier external/internal reference can improve accuracy. **Always convert with the actual Vref**, or your volts are wrong.
- **Sample rate:** one conversion takes time (~100 µs on the Uno). Reading too often, or a signal that changes too fast, distorts the result. Slow quantities (temperature, a knob) are easy; audio-rate signals depend on the chip's sampling ability.

Different boards, different depths:

| Board | ADC bits | Full-scale reading | Logic |
|---|---|---|---|
| Arduino Uno | 10-bit | 0–1023 | 5 V |
| ESP32 | 12-bit | 0–4095 | 3.3 V |
| Pi Pico | 12-bit | 0–4095 | 3.3 V |

> ✦ **Key point:** ADC = quantization. reading = round(Vin/Vref × (levels−1)). To recover volts, multiply back by Vref and divide by full scale. More bits is finer, but the conversion logic is unchanged.
