## 核心讲义

### 一根线也能传数据

DHT22(又名 AM2302)是一颗温湿度传感器,只用**一根数据线**就能把温度和湿度都传给单片机。它内部已经把模拟量转成了数字,不需要你接 ADC——但代价是,它有一套**自定义的单总线时序协议**,收发双方要在这根线上按约定的高低电平时长「打摩斯电码」。

### 40 位数据帧与校验和

单片机先拉低数据线一段时间发出「请求」,DHT22 响应后,一次吐出 **40 个比特**:

| 段 | 位数 | 含义 |
|---|---|---|
| 湿度高字节 | 8 | 湿度整数部分 |
| 湿度低字节 | 8 | 湿度小数部分 |
| 温度高字节 | 8 | 温度整数部分 |
| 温度低字节 | 8 | 温度小数部分 |
| 校验和 | 8 | 前四字节之和的低 8 位 |

每一位是 0 还是 1,取决于高电平持续的**时间长短**(短=0,长=1)。最后一个字节是**校验和**:把前四个字节相加,低 8 位应当等于它。对不上就说明这次读数受了干扰,应丢弃。

### 用库读取

这套时序用手写很折腾,实际都用现成库(如 DHT sensor library):

```cpp
#include <DHT.h>
#define DHTPIN 2
#define DHTTYPE DHT22
DHT dht(DHTPIN, DHTTYPE);

void setup() { Serial.begin(9600); dht.begin(); }
void loop() {
  float h = dht.readHumidity();
  float t = dht.readTemperature();
  if (isnan(h) || isnan(t)) { Serial.println("read failed"); }
  else {
    Serial.print(t); Serial.print("C  ");
    Serial.print(h); Serial.println("%");
  }
  delay(2000);   // DHT22 至少间隔 2 秒
}
```

### 两个常见坑

- **采样别太快:** DHT22 每次读取至少间隔 **2 秒**,读太频繁会得到旧值或读失败。
- **上拉电阻:** 数据线通常需要一个约 **10 kΩ 上拉**到 Vcc(很多模块板已内置)。

> ✦ **要点:** DHT22 用**单总线时序**在一根线上串出 40 位数据(湿度+温度+校验)。原理值得理解,但实战直接用库。记住:**2 秒间隔、数据线上拉、校验和验数据**。这类「自带协议的数字传感器」在嵌入式里非常普遍。
