## 核心讲义

### 带 Wi-Fi 和蓝牙的主控

ESP32 是一颗**双核 32 位**微控制器,最大的卖点是**内置 Wi-Fi 和蓝牙**。当你的项目需要联网——上传数据到云端、做个网页控制台、接入物联网平台——ESP32 几乎是默认选择。它的主频、内存、外设都远超 Arduino Uno。

### 3.3V 逻辑:最重要的一条注意

和 Uno 最关键的区别:**ESP32 是 3.3V 逻辑,不是 5V**。它的引脚**不能承受 5V**,把 5V 传感器的输出直接接到 ESP32 的输入脚,可能损坏芯片。与 5V 器件互连时,需要**电平转换**(电平转换模块,或简单分压见 CF3)。供电通常走 USB(板载稳压到 3.3V),不要用 5V 直接灌 3.3V 的脚。

### 引脚可以「重映射」

Uno 的 I²C、SPI、串口都固定在特定引脚。ESP32 内部有一个 **GPIO 矩阵**,大多数外设功能可以**灵活映射到不同引脚**——`Wire.begin(SDA, SCL)` 里你可以指定用哪两个脚做 I²C。但要注意有些脚有特殊用途(启动时的 strapping 脚、只能输入的脚、接了 Flash 的脚),接线前查一下该开发板的引脚图。

### Wi-Fi 与深睡眠

连 Wi-Fi 只需几行:

```cpp
#include <WiFi.h>
void setup() {
  Serial.begin(115200);
  WiFi.begin("SSID", "password");
  while (WiFi.status() != WL_CONNECTED) { delay(300); Serial.print("."); }
  Serial.println(WiFi.localIP());
}
void loop() {}
```

无线很耗电。电池供电的项目常用**深睡眠(deep sleep)**:大部分时间让芯片休眠到几十微安,定时或被引脚唤醒,醒来采集/上报后再睡。这是电池能撑几个月的关键。

> ⚠ **要点:** ESP32 = 更强的双核 MCU + 无线。用它前先记牢:**3.3V 逻辑、引脚勿超压、外设可重映射**。联网省电靠深睡眠。它与 Uno 的编程接口相似(Arduino 框架),迁移代码时主要改的是引脚电压和外设初始化。
