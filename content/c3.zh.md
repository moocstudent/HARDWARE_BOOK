## 核心讲义

### 四根线,又快又全双工

当你要驱动彩色显示屏、读写 SD 卡这类**大流量**器件时,I²C 的两线速度可能不够。**SPI(串行外设接口)**用**四根线**换来了高速和**全双工**(收发同时进行),是三大总线里最快的。

- **SCK(Serial Clock):** 时钟,主机产生
- **MOSI(Master Out, Slave In):** 主机 → 从机 的数据
- **MISO(Master In, Slave Out):** 从机 → 主机 的数据
- **CS / SS(Chip Select):** 片选,主机用它点名某个从机

和 I²C 一样有时钟线,所以 SPI 也是**同步**的;但它用**独立的收发两根线**,能一边发一边收,速度也快得多。

### 移位寄存器:发和收是同一件事

SPI 的底层模型是**两个首尾相接的移位寄存器**(主、从各一)。每来一个时钟沿,双方各移出一位、移入一位——所以**发送和接收是同时发生**的。哪怕你只想读数据,也得同时发出一个「哑」字节来产生时钟、把从机的数据「顶」出来:

```cpp
byte received = SPI.transfer(0x00);  // 发 0x00,同时收回一字节
```

### 片选:靠 CS 选择多个从机

I²C 靠地址区分器件,SPI 则靠**片选线**:每个从机单独一根 CS。主机要和谁通信,就把它的 CS 拉低(选中),通信完再拉高(释放)。多几个从机就多几根 CS——这也是 SPI 的代价:器件越多,占用引脚越多。

```cpp
#include <SPI.h>
void setup() {
  pinMode(10, OUTPUT);          // CS
  SPI.begin();
}
void write_reg(byte reg, byte val) {
  digitalWrite(10, LOW);        // 选中从机
  SPI.transfer(reg);
  SPI.transfer(val);
  digitalWrite(10, HIGH);       // 释放
}
```

### 时钟极性与相位(SPI 模式)

SPI 有四种**模式**,由两个参数决定:**CPOL**(时钟空闲时是高还是低)和 **CPHA**(在时钟的第一个还是第二个边沿采样)。主从两端的模式必须一致,否则数据会错位。数据手册会写明该器件用哪种模式,照着 `SPISettings` 配即可。

### 三大总线一览

| | UART | I²C | SPI |
|---|---|---|---|
| 线数 | 2 | 2 | 4+ |
| 时钟 | 无(异步) | 有 | 有 |
| 多从机 | 否 | 靠地址 | 靠片选 CS |
| 速度 | 低-中 | 中 | 高 |
| 适合 | 调试/点对点 | 多个慢速小器件 | 显示屏/SD卡等高速 |

> ✦ **要点:** SPI = 四线、同步、全双工,靠**片选**管理多从机,速度最高。核心直觉是「移位寄存器:发即是收」。用之前对齐 **SPI 模式(CPOL/CPHA)**。器件多、要引脚多是它的取舍。
